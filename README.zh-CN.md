## 项目概览

这是一个面向 国服皇室战争 自动对战的视觉驱动控制项目，目标是在安卓模拟器或安卓设备上完成对局识别、卡牌状态跟踪、资源节奏管理与自动出牌。

项目当前围绕两套内置卡组展开：
- `Elixir Golem` 自动打法。
- `Royal Recruits` 自动打法。

本项目采用mumu模拟器模拟皇室战争国服，需要使用者自行配置环境

## 整体目标与用途

- 让程序能够自动进入或接管战斗，并在一整局内持续执行出牌逻辑。
- 让程序能够识别当前对局中的卡牌、手牌顺序、下一张牌与战斗阶段。
- 让程序能够根据卡组和当前阶段管理资源节奏，避免无效出牌或资源失控。
- 让程序能够支持不同运行模式，包括正常匹配循环、直接接管当前战斗、调试采样与离线评估。
- 让程序能够为识别与策略调优提供可重复的记录、样本和验证手段。

## 工作库与作用

- `opencv-python`：用于截图处理、区域裁剪、模板匹配、颜色与边缘特征分析。
- `numpy`：用于图像数组运算、掩码计算和数值统计。
- `uiautomator2`：用于连接安卓设备或模拟器，并执行截图、点击和应用前台控制。
- `Pillow`：用于文本区域的字体模板生成与字符级识别辅助。
- Python 标准库：用于命令行参数、线程、时间控制、进程调用、文件路径和运行状态管理。

## 模块划分

### 设备与运行入口

- 负责程序启动、参数解析、卡组选择和运行模式切换。
- 负责连接安卓设备或模拟器，并为后续视觉识别与点击操作提供统一入口。
- 负责将“正常对战循环”和“直接接管当前战斗”两种运行方式串起来。

### 屏幕感知与状态判定

- 负责从设备截图中判断战斗是否开始、是否结束，以及当前是否仍处于有效对战界面。
- 负责读取战斗时间与阶段信息，将对局分为单倍、双倍、三倍等节奏区间。
- 负责为主流程提供稳定的“开始/结束/阶段变化”信号。

### 卡牌识别与牌序跟踪

- 负责从卡牌区域识别当前手牌、待抽卡牌与下一张牌。
- 负责维护卡牌循环状态，持续修正“已知、未知、待确认”的牌序信息。
- 负责在出牌前后做复核，降低识别误差对战斗流程的影响。

### 对战流程与出牌策略

- 负责把识别到的卡牌状态转换为实际的出牌动作。
- 负责按照不同卡组的节奏模板执行分阶段打法。
- 负责在关键时机处理冠军技能、资源压力和节奏切换。
- 负责在出牌后同步更新循环状态和资源状态，保证后续决策一致。

### 资源与节奏管理

- 负责把卡牌费用、技能费用与当前资源消耗关联起来。
- 负责按战斗阶段估算资源恢复速度，并在低资源或临界资源时避免越界操作。
- 负责在战斗开始、结束和阶段切换时重置或结算资源状态。

### 调试、采样与评估工具

- 负责采集战斗截图、关键区域样本和调试记录。
- 负责对识别效果做离线评估与效率对比。
- 负责在启动前检查环境、分辨率和设备连接是否可用。

## 方法总览

- 采用“设备截图 + 视觉识别 + 自动点击”的闭环控制方式。
- 采用模板匹配、区域裁剪、颜色/边缘/纹理组合判断来识别卡牌和战斗信息。
- 采用多帧观察和结果投票来提升识别稳定性，降低单帧噪声影响。
- 采用循环状态机来表示手牌、弃牌堆和待抽牌的关系，并通过出牌前后复核修正状态。
- 采用战斗阶段驱动的资源管理思路，把节奏、费用和技能释放统一到同一套运行状态中。
- 采用分卡组策略的方式把通用流程和具体打法分离，便于扩展新的卡组方案。

## 典型战斗场景

- 正常自动连局：等待匹配、识别开局、执行整局打法、等待结算后进入下一局。
- 直接接管当前战斗：跳过等待流程，直接进入正在进行的对局并继续执行自动打法。
- 卡组专项打法：围绕固定卡组在单倍、双倍、三倍阶段切换不同节奏。
- 识别调试：采集截图、检查卡牌识别效果、分析误判与未知牌。
- 离线评估：对历史截图样本做准确率和耗时评估，用于比较识别方案。

## 总结

这个项目的核心不是单一的“自动点按”，而是一套围绕《皇室战争》战斗界面的完整自动化系统：先感知状态，再维护牌序与资源，再按卡组节奏执行操作，最后通过调试与评估工具不断校正识别和策略。
