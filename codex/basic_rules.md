# 基础规则研究报告

适用范围：`elixir_golem` / `royal_recruits` 两套脚本研究的通用基础约束。  
目标：把可程序化的通用规则和经验启发拆开，供后续 deck-specific 研究直接复用。

## 一、硬规则

### 1. 费用与溢出

- 费用值不能为负，放牌前必须确认当前圣水 `>=` 牌费。
- 必须显式防止圣水溢出。若当前圣水接近上限，优先用低风险、低承诺单位或空放型循环牌消耗。
- 任何“等一等再放”的逻辑都要有上限，不能无限等待导致持续溢出。
- 若处于对方压制期，脚本应优先保证防守响应权，而不是为了省费继续贪圣水。

### 2. 兵种放置约束

- 放置必须满足最小可用距离、攻击范围、桥位/后置位合法性等基础几何约束。
- 不能把关键高价值单位直接丢进无收益火力区，除非这是明确的换费或强行接战决策。
- 防守单位优先落在能够同时覆盖目标与塔的区域；若无法同时覆盖，优先覆盖高威胁目标。
- 远程单位、脆皮单位、持续输出单位要避免与对方主力直线重叠，防止被法术或溅射“一锅端”。
- 群体单位与高价值支援单位之间要保持最低安全间隔，避免被同一张法术同时处理。

### 3. 位置与站位

- 所有站位都应围绕“拉扯、分离、延迟、保留输出时间”来设计。
- 防守拉扯优先级高于追求极限贴脸；能把目标拉到中线就不要放到同侧深处。
- 建筑、坦克、前排、后排要分层摆放，禁止无意义堆叠。
- 任何会被地面溅射或法术覆盖的站位，都应视为高风险站位。

### 4. 循环与出牌

- 脚本必须维护手牌循环状态，避免在关键防守窗口把核心解牌打空。
- 若当前手牌无法覆盖已识别威胁，应优先保留解牌，不要为了“继续循环”而破坏防守完整性。
- 任何推进动作都必须带有“可回收成本”判断：进攻失败后是否还能及时回防。

## 二、经验启发

### 1. 圣水再生与防溢出

- `Estimate`：常规阶段圣水自然回复约为每 `2.8s` 1 点；双倍、三倍阶段会显著加快。  
- `Heuristic`：脚本不应按固定节拍盲目下牌，而应按“当前圣水 + 下一个可用牌费 + 敌方压迫强度”联动决策。
- `Heuristic`：接近满费时，优先寻找低风险泄洪手段，而不是强行等待高价值连招。
- `Heuristic`：在双倍/三倍阶段，溢出风险高于单倍阶段，脚本需要更积极地做费用回收与节奏切换。

### 2. 兵种角色

- 前排/坦克：吸收火力、创造推进时间；其价值不只在伤害，也在“替后排争取生存帧数”。
- 后排/输出：主要提供稳定伤害，最怕被提前暴露到法术或直线斩杀区。
- 拉扯/控场：主要价值是改变敌方行进路线与攻击目标，给塔和主力争取输出窗口。
- 反打/收割：主要用于吃掉对方过度投入后的残血资源，避免脚本把资源继续堆进已失败线路。

### 3. 典型站位偏好

- `Heuristic`：面对地面推进，优先中线拉扯而不是同路硬顶。
- `Heuristic`：面对桥头压制，优先用“分层防守”拆开对方资源，不要让多个关键单位同时吃同一波伤害。
- `Heuristic`：后置部署更适合蓄势型推进，桥位部署更适合抢节奏与打断对方铺场。
- `Heuristic`：对空与对地职责要尽量分离，避免一个单位同时承担过多职责导致被针对后全线失效。

### 4. 单倍 / 双倍 / 三倍阶段的人类倾向

- 单倍圣水：玩家更谨慎，试探较多，常见行为是小费试探、低承诺换牌、观察对方解牌。
- `Estimate`：双倍圣水：进攻频率提升，更多玩家会尝试把防守转为反打，容错空间下降。
- `Estimate`：三倍圣水：爆发式投入增多，短时间内出现高密度交换，常见“先出手再问结果”的倾向。
- `Heuristic`：脚本在单倍阶段应偏保守、重信息收集；在双倍/三倍阶段应偏节奏控制、重费用调度。

## 三、脚本相对人类的强弱

### 优势

- `Heuristic`：脚本能稳定执行费用阈值、站位间距、循环保留等规则，不会因情绪或贪心而失控。
- `Heuristic`：脚本可以持续跟踪手牌、敌方已知资源、当前阶段，做重复且一致的决策。
- `Heuristic`：脚本适合处理明确模式化局面，例如固定桥压、固定拉扯、固定防守模板。

### 弱点

- `Heuristic`：脚本通常不擅长处理模糊信息和隐性意图，例如诱骗、虚晃、读牌、临时改线。
- `Heuristic`：脚本容易被“低频高收益”的人类非标准打法干扰，例如故意空过、反常沉底、延迟交牌。
- `Heuristic`：脚本如果过分依赖固定模板，会在被针对后出现重复失误。

## 四、面向下游研究的可复用约束

- 约束 1：所有决策先检查 `圣水阈值`、`手牌可用性`、`站位合法性`，任一不满足则禁止出牌。
- 约束 2：建立“关键解牌保留表”，防守窗口内不得因循环需求消耗核心牌。
- 约束 3：任何推进必须标记为 `可回防` 或 `不可回防`；后者只允许在明确优势或终局条件下执行。
- 约束 4：对每个单位维护 `角色标签`，至少区分前排、输出、控场、拉扯、收割、补位。
- 约束 5：对每个站位维护 `风险标签`，至少区分桥位、后置位、中线位、同侧深位、可被法术覆盖位。
- 约束 6：对每个阶段维护不同的决策权重，单倍重信息，双倍重转换，三倍重爆发与费用回收。
- 约束 7：遇到不确定局面时，优先采用低承诺动作获取信息，而不是一次性投入全部资源。
- 约束 8：所有“经验规则”必须允许被局部覆盖，避免硬编码成不可调整的死规则。

## 五、可直接复用的简化结论

- `Hard Rule`：不溢费，不空防，不让核心解牌在关键窗口失位。
- `Hard Rule`：站位先合法，再谈收益。
- `Heuristic`：单倍偏稳，双倍偏转，三倍偏快，但任何阶段都不能放弃基本防守完整性。
- `Heuristic`：脚本要比人更稳定，但必须接受人类的非标准干扰，因此需要保留低承诺探测与局部改线能力。
